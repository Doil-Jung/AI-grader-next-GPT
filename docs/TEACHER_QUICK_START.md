# AI 채점기 by GPT — 교사용 빠른 시작

## 1. 처음에는 가상 샘플로 확인하세요

상단 `도움말`에서 `가상 샘플 프로젝트 만들기`를 누르면 실제 학생 자료와 API 호출 없이 전체 화면을 둘러볼 수 있습니다. 샘플에는 가상 학생 5명, 독립 채점 2회, 수동 점수와 교사 확정 점수가 들어 있습니다.

## 2. 모든 프로젝트의 공통 순서

1. **개요**: 현재 준비 상태와 다음 작업을 확인합니다.
2. **평가기준**: 공식 기준을 입력·추출하거나 자동 초안을 만든 뒤 교사가 승인합니다.
3. **학생·답안**: 명렬을 저장하고 통합 스캔을 분할하거나 학생별 파일을 연결합니다.
4. **AI 채점**: 실행 조건과 예상 요청 수를 확인하고 독립 채점 2회를 실행합니다.
5. **검토·확정**: AI 평균·표준편차·수동 점수·차이를 확인하고 교사가 최종 점수를 확정합니다.
6. **내보내기·분석**: 확정 점수표와 신뢰도 분석을 Excel로 저장합니다.

AI 점수는 교사 확정 점수를 자동으로 덮어쓰지 않습니다. 새 회차나 수동 점수가 생겨도 기존 확정값은 보존되고 재확인 대상으로 표시됩니다.

## 3. 보고서·수행평가

1. 공식 루브릭이 있으면 평가기준 화면에서 파일을 올려 추출합니다.
2. 루브릭이 없으면 평가 목적을 짧게 입력해 초안을 만듭니다.
3. 평가 항목·척도·핵심 기준을 확인하고 기준 버전을 승인합니다.
4. `번호. 이름.pdf`처럼 번호로 시작하는 학생 파일을 연결합니다.
5. 독립 채점 2회를 실행합니다.
6. 필요하면 `선채점 후 기준화`로 실제 답안에서 대안 표현과 경계 사례를 찾아 새 기준 초안을 만듭니다.
7. 검토·확정 뒤 첫 시트가 `확정 점수표`인 분석 Excel을 저장합니다.

## 4. 정기고사 서술형

1. 문제지와 채점기준표가 있으면 함께 올립니다.
2. 기준표가 없으면 문제지만 올려 모범답안·부분점 기준 초안을 생성합니다.
3. 대문항과 소문항, 배점 합계, 답안 유형을 확인합니다.
4. 학생 명렬을 실제 스캔 순서대로 입력합니다.
5. 한 반 통합 PDF를 올리고 학생당 쪽 수 또는 학생별 시작 쪽을 입력합니다.
6. 반드시 분할 미리보기에서 학생별 범위를 확인한 뒤 PDF를 생성합니다.
7. 독립 채점 2회를 실행하고 소문항별 차이가 큰 답안을 우선 검토합니다.
8. 교사 확정 뒤 소문항 점수와 대문항 합계가 포함된 Excel을 저장합니다.

## 5. 대회·탐구 심사

1. 보고서 평가와 같은 방식으로 원 평가를 완료합니다.
2. `내보내기·분석`에서 원 평가 회차를 선택합니다.
3. 행을 드래그해 최종 순서를 정합니다.
4. 동점은 `위와 공동순위`를 선택합니다.
5. 허용 최종 점수표로 자동 배정한 결과를 확인합니다.
6. 자동 점수를 직접 바꾸려면 예외 점수와 사유를 함께 입력합니다.
7. 교사 확정 뒤 Excel을 저장합니다.

원 평가점수와 원 평가 항목점수는 바뀌지 않습니다. 최종 순위와 배정점수는 별도 파일과 변경 이력으로 관리됩니다.

## 6. 개발 피드백 묶음

`익명 개발 피드백 ZIP`은 추가 설문 없이 기존 채점 기록으로 만듭니다. 학생 답안, 문제지, 정답지, 루브릭 원본은 포함하지 않습니다. 실제 학생과 가상 ID의 대응표는 프로젝트의 `private/feedback_mappings`에만 저장됩니다.

자동 비식별화가 모든 비정형 개인정보를 보장할 수는 없으므로 외부 전달 전 안내와 표 내용을 한 번 확인하세요.

## 7. 데이터 위치와 이동

- 프로젝트 설정·학생 자료·결과: 앱 폴더의 `projects`
- 생성 Excel과 피드백 ZIP: 각 프로젝트의 `results` 또는 `exports`
- 실제↔가상 학생 대응표: 각 프로젝트의 `private/feedback_mappings`
- API 키: 앱 폴더의 `.api_keys.json`

기기를 옮길 때는 프로젝트 폴더 전체를 동기화합니다. `.git`, 패키지 설치 폴더, 브라우저 프로필과 빌드 임시 파일은 프로젝트 데이터가 아닙니다.
