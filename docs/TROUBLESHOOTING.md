# AI 채점기 오류 해결 안내

## API 키 또는 로그인 오류

- 상단 `API 키`에서 현재 프로젝트의 공급자 키가 설정되어 있는지 확인합니다.
- 키를 바꿔도 이미 시작한 회차의 공급자·모델은 바뀌지 않습니다.
- 모델을 바꾸려면 기존 회차 이어하기가 아니라 `새 독립 회차`로 시작합니다.

## 쿼터·결제·요청 한도

- 쿼터 오류가 난 회차의 성공 학생 결과는 보존됩니다.
- 결제·일일 한도를 확인한 뒤 같은 조건이면 실패 학생만 재시도합니다.
- 다른 모델로 바꿀 때는 새 회차를 만듭니다.
- 화면의 비용은 거친 예상치이며 공급자 청구 내역이 최종 기준입니다.

## AI 응답 형식 또는 일시적 오류

- 같은 회차의 `실패 학생 재시도`를 먼저 사용합니다.
- 계속 실패하면 파일이 열리는지와 PDF 손상 여부를 확인합니다.
- 오류가 특정 학생에게만 반복되면 해당 PDF 쪽 수와 회전 방향, 스캔 선명도를 확인합니다.

## 통합 스캔 분할이 맞지 않음

- 명렬 순서가 실제 스캔 순서와 같은지 확인합니다.
- 모든 학생의 쪽 수가 같으면 학생당 쪽 수를 사용합니다.
- 학생마다 다르면 학생별 시작 쪽을 입력합니다.
- 실행 전에 분할 미리보기의 시작·끝 쪽을 반드시 확인합니다.
- 원본 통합 PDF는 수정되지 않으므로 설정을 고쳐 다시 분할할 수 있습니다.

## 학생 파일 연결이 틀림

- 학생·답안 화면에서 이름 불일치와 여러 파일 상태를 확인합니다.
- 파일별 연결 학생을 직접 바꿀 수 있습니다.
- 잘못된 수동 연결이 많으면 전체 자동 연결로 되돌린 뒤 다시 지정합니다.
- 프로젝트 내부 경로는 상대경로로 저장되므로 프로젝트 폴더 전체가 이동했는지 확인합니다.

## 문항이 누락되거나 점수가 이상함

- 문제 인식 결과의 대문항·소문항 수와 배점을 먼저 확인합니다.
- 소문항이 있는 대문항은 직접 채점하지 않고 소문항 합계로 계산됩니다.
- 자율 선채점은 실제 답안의 다양한 풀이를 넓게 인정할 때 유리합니다.
- 공식 기준을 엄격히 적용하려면 기준 전달 방식을 `공식 기준 엄격 적용`으로 바꿉니다.
- 기준 변경은 새 버전으로 승인한 뒤 새 독립 회차에 적용합니다.

## 회차 간 점수 차이가 큼

- 표준편차, 최대 차이, AI 확인 필요와 낮은 확신도 표시를 함께 봅니다.
- 전체를 무조건 다시 채점하기보다 추가 회차 추천 학생만 먼저 실행합니다.
- 수동 점수가 있으면 AI-수동 차이가 큰 학생·문항을 우선 확인합니다.
- 반복되는 경계 사례는 `선채점 후 기준화`에서 새 기준 초안으로 정리합니다.

## 확정 점수가 자료 변경으로 표시됨

- 교사 확정 뒤 새 회차 또는 수동 점수가 추가·변경된 상태입니다.
- 기존 확정점수는 보존되어 있습니다.
- 검토·확정에서 새 자료와 비교한 뒤 다시 승인하거나 기존 결정을 유지합니다.

## 대회 순위표를 내보낼 수 없음

- 먼저 자동 배정 미리보기를 만들고 교사 확정해야 합니다.
- 자동 점수와 다른 예외 점수에는 사유가 필요합니다.
- 공동순위 팀은 같은 최종 점수를 사용합니다.
- 확정 뒤 원 평가 회차가 변경되면 순위 조정안을 다시 확정해야 합니다.

## 익명 피드백 묶음 생성 실패

- 알려진 학생 이름, 학교명, 연락처, 주소, 경로 또는 인증정보 흔적이 발견되면 저장을 중단합니다.
- AI 근거나 오류 로그에 특이한 개인 식별 문구가 있는지 확인합니다.
- 실패 시 ZIP과 실제↔가상 대응표는 새로 남기지 않습니다.
- 원본 자료가 필요한 문제 사례는 표준 묶음에 임의로 추가하지 마세요.

## 앱이 저장하지 못함

- 상단 `도움말`의 `환경·프로젝트 진단`을 실행합니다.
- 프로젝트 폴더가 원드라이브에서 동기화 충돌 또는 읽기 전용 상태인지 확인합니다.
- 앱을 여러 창에서 동시에 열어 같은 프로젝트를 수정하지 않습니다.
- 기존 `projects` 폴더를 백업한 뒤 패키지를 다시 설치합니다.
